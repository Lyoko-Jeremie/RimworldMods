# RimWorld 万能制造机 (OmniCrafter) Mod 设计文档

## 1. 核心概述
本 Mod 旨在添加一个“万能工作台/物质合成机”。该工作台不需要小人手动工作，而是通过瞬间消耗电网中储存的电量（基于物品价值与品质），自动生产游戏中的**任何物品**（包括材料、武器、食物、打包好的建筑，甚至超凡科技物品）。

---

## 2. UI 界面架构设计
界面继承自 `Window` 类，采用左、中、右三分栏的经典复杂 Mod 布局，注重信息密度与操作效率。

### 2.1 顶部：全局控制台
* **搜索栏 (Search Bar)**：
    * 支持中英文双语搜索。
    * 搜索逻辑同时匹配物品的显示名称 (`label`) 和内部代码名 (`defName`)。
    * 支持拼音首字母模糊搜索（进阶优化）。（暂不实现）
    * 搜索采用缓存，提高搜索响应速度。

### 2.2 左侧：导航树 (Navigation Tree)
* **收藏夹 (Favorites)**：玩家标记的常用物品，常驻最上方。
* **最近生产 (Recent)**：自动记录并显示最近生产的 5~10 个物品。
* **分类树 (Categories)**：读取原版的 `ThingCategoryDef`（如武器、服装、资源等），点击分类以过滤中间列表。

### 2.3 中间：物品展示区 (Item View)
* **排序器 (Sorter)**：位于列表顶部，支持按“市场价值”、“重量”、“名称”排序。
* **筛选器 (Mod Filter)**：支持按照物品所属的Mod进行过滤（如只显示原版物品，或只显示某个特定Mod的物品）。
* **网格/列表 (Grid/List)**：展示当前分类或搜索结果下的物品图标。
* **性能优化**：必须使用视口裁剪渲染（仅渲染当前滚动条可视区域内的图标），并缓存当前分类的 `List<ThingDef>`。

### 2.4 右侧：详情与生产配置 (Details & Configuration)
* **物品预览**：大图标展示、名称、基础描述、核心属性、收藏按钮。
* **材质选择器 (Stuff Selector)**：
    * 仅当物品具有 `MadeFromStuff` 属性时显示。
    * **必须过滤**：只列出符合该物品 `stuffCategories` 要求的材质。
* **品质选择器 (Quality Selector)**：
    * 允许选择极劣~传奇品质。
    * **电力乘数联动**：品质越高，所需消耗的电力呈指数级上升（动态显示在下方）。
* **生产模式与数量**：
    * `生产 X 个`（单次生产）。
    * `生产直到拥有 X 个`（维持库存模式）。
* **输出位置设置**：
    * `掉落在周围`：工作台旁边，超出堆叠上限时向外围扩散。
    * `发送至最佳储存区`：自动寻找距离最近且优先级最高的合适储存区。
* **操作区**：
    * **“开始生产”按钮**：并在旁边清晰显示【本次生产预估耗电量 / 当前电网总电量】。

## 3. 核心后端逻辑 (C# 开发要点)

### 3.1 物品过滤机制 (黑名单/白名单)
为了防止游戏崩溃，必须在缓存阶段过滤掉无法正常生成的 `ThingDef`：
* 必须剔除：子弹/抛射物 (`Projectile`)、特效 (`Mote`)、天空坠落物 (`Skyfaller`)。
* 必须剔除：带有 `Base` 且无法实例化的抽象模板类。
* 必须剔除：不可被摧毁的地图机制物体 (`destroyable == false`)。
* **植物的特殊处理**：如果是植物分类，生成的应该是其“收获产物”（如玉米），而不是直接在地上生成一棵活体植物对象（会因没有土壤而报错）。

### 3.2 耗电与结算系统
* **电网识别**：通过 `this.GetComps<CompPowerTrader>().PowerNet` 获取当前电网，遍历所有的 `batteryComps`。
* **扣费逻辑**：按比例或按顺序从电池中扣除相应的 `Wd`（瓦特天）。
* **前置拦截**：在玩家点击“生产”瞬间，判断总电量是否足够。不足则提示“电网储能不足”，并阻断生成动作。

### 3.3 物品生成与状态处理 (`GenSpawn.Spawn`)
* **打包建筑**：如果选择的是建筑 (`Building`)，必须调用 `MinifyUtility.MakeMinified()` 将其打包为 `MinifiedThing`，然后再生成。
* **艺术描述**：如果生成的品质是卓越或更高，必须调用 `ArtGeneration` 相关的代码，为其生成艺术描述。
* **溢出处理**：生成大量低堆叠上限物品（如木材）时，需调用原版的掉落逻辑（寻找附近可用空地），防止同格堆叠数量违规。

### 3.4 自动化“维持库存”逻辑性能
对于“生产直到全局有 X 个”的功能：
* 采用与原版游戏界面左侧显示物品数量统计同样的计算方法进行计算。
* **禁止每帧检测**：绝不能在 `Tick()` 中每帧遍历全图物品。
* **Tick 优化**：使用 `TickRare()`（每 250 Ticks 运行一次）或自定义计数器（每 1000 Ticks，约游戏内 1 小时检查一次）来扫描全图特定物品的库存总数，数量不足时自动扣电生成。

## 4. 开发步骤建议 (Roadmap)
1. **阶段一（骨架）**：编写 XML，让工作台出现在游戏中，耗电，并绑定一个点击能弹出空白 C# 窗口的按钮。
2. **阶段二（核心渲染）**：实现中间的网格列表和顶部的搜索框。完成最基础的生成代码（点击图标直接在脚下生成原木）。
3. **阶段三（属性扩展）**：实现右侧详情栏，重点攻克 **材质选择** 和 **品质选择** 这两个深水区。
4. **阶段四（能源与经济）**：编写读取电池储量和扣除电量的逻辑，将物品价值换算为耗电量。
5. **阶段五（体验打磨）**：完成收藏夹、分类树、自动循环生产逻辑以及各种异常情况的报错拦截。
6. **外源异常捕获**：注意在对物品进行遍历、筛选、搜索、匹配、显示图标等时，需要考虑到某些来自其他Mod的物品可能不完全符合游戏的定义，例如缺少图片、缺少label、有非法字符等等。
